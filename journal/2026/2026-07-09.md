## What I Learned

### Measuring CPU Performance

Today I learned that CPU performance is not determined only by clock frequency.

A common metric is **CPI (Cycles Per Instruction)**.

One of the major contributors to CPI is memory access latency.

Reducing the time spent waiting for memory is one of the key goals of modern processor design.

---

### Cache Mapping Strategies

I studied three cache organizations.

- Direct-Mapped Cache
- Set-Associative Cache
- Fully Associative Cache

I learned that these designs are trade-offs between:

- Hardware complexity
- Lookup latency
- Cache miss rate

Modern processors usually adopt **Set-Associative Cache** because it provides a good balance between performance and implementation cost.

---

### Cache Miss Optimization

I also learned why cache conflicts occur.

When multiple memory blocks compete for the same cache line, conflict misses become frequent.

Increasing associativity can significantly reduce these conflicts.

## Biggest Insight

Today I corrected several misunderstandings.

For example:

- Direct-Mapped Cache still compares the tag.
- A cache is organized as **Sets**, and each set contains multiple **Ways**.
- Write Back updates memory when a dirty cache line is evicted, rather than on every access.

These details helped me build a more accurate mental model of cache behavior.
