## What I Learned

Today I focused on one of the most fundamental concepts in operating systems: **Virtual Memory**.

I learned that programs never access physical memory directly. Instead, they operate on virtual addresses, which are translated by the MMU into physical addresses before any cache or memory lookup occurs.

I also learned the main motivations behind virtual memory:

- Process isolation
- Memory protection
- Demand paging
- Memory sharing

---

### Address Translation

I now understand the complete memory access path:

```
Program
    ↓
Virtual Address
    ↓
TLB
    ↓
Page Table
    ↓
Physical Address
    ↓
L1 / L2 / L3 Cache
    ↓
Memory
    ↓
Disk (only if a Page Fault occurs)
```

This helped me connect several concepts that previously felt unrelated.

---

### TLB

One of today's biggest insights was understanding the role of the TLB.

The TLB is **not a data cache**.

Instead, it caches **virtual-to-physical address translations**, avoiding repeated page table lookups.

---

## Biggest Insight

Today I realized that computer architecture is built as a hierarchy of caches:

- CPU Cache caches data.
- TLB caches address translations.
- Memory caches disk data through demand paging.

Every layer exists to reduce the latency of the layer below it.
